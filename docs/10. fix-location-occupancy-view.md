# Fix: Location Occupancy View Not Updating

**Issue**: Pallet occupancy shows 0 or 100% even after assigning pallets to locations

**Root Cause**: The `location_occupancy` view only counted pallets with status `'STORED'`, but pallets are set to `'PUTAWAY_PENDING'` when assigned to a location, so they weren't counted.

---

## Quick Fix (Run This SQL)

If you have access to the database, run this command to fix the view:

```sql
-- Drop and recreate the location_occupancy view to include PUTAWAY_PENDING status
DROP VIEW IF EXISTS public.location_occupancy CASCADE;

CREATE VIEW public.location_occupancy AS
SELECT
  wl.id AS location_id,
  wl.location_code,
  wl.warehouse_id,
  wl.company_id,
  wl.max_stacks,
  wl.grid_zone,
  wl.grid_row,
  wl.grid_column,
  wl.status,
  COUNT(p.id) AS current_occupancy,
  wl.max_stacks - COUNT(p.id) AS available_slots,
  CASE
    WHEN COUNT(p.id) = 0 THEN 'EMPTY'
    WHEN COUNT(p.id) < wl.max_stacks THEN 'PARTIAL'
    WHEN COUNT(p.id) >= wl.max_stacks THEN 'FULL'
  END AS occupancy_status
FROM public.warehouse_locations wl
LEFT JOIN public.pallets p
  ON p.location_id = wl.id
  AND p.status IN ('PUTAWAY_PENDING', 'STORED')
GROUP BY wl.id, wl.location_code, wl.warehouse_id, wl.company_id, wl.max_stacks,
         wl.grid_zone, wl.grid_row, wl.grid_column, wl.status;
```

## What Changed

### Before
```sql
LEFT JOIN public.pallets p ON p.location_id = wl.id AND p.status = 'STORED'
```

**Problem**: Pallets with status `'PUTAWAY_PENDING'` weren't counted

### After
```sql
LEFT JOIN public.pallets p
  ON p.location_id = wl.id
  AND p.status IN ('PUTAWAY_PENDING', 'STORED')
```

**Solution**: Now counts both statuses

## Pallet Status Lifecycle

```
RECEIVED
   ↓
User selects location & confirms
   ↓
PUTAWAY_PENDING ← NOW COUNTED IN OCCUPANCY
   ↓
Operator places pallet physically
   ↓
STORED ← COUNTED IN OCCUPANCY
   ↓
Shipped out or moved
   ↓
SHIPPED / DEFECT
```

The occupancy view now correctly counts pallets at any point after assignment (PUTAWAY_PENDING or STORED).

## Troubleshooting

### Issue: View still shows 0 occupancy

1. Verify the SQL ran without errors
2. Check pallet records exist:
   ```sql
   SELECT id, location_id, status FROM pallets LIMIT 5;
   ```
3. Verify pallets have location_id filled:
   ```sql
   SELECT COUNT(*) FROM pallets WHERE location_id IS NULL;
   ```
   Should be low or 0

### Issue: "View already exists" error

Just run with `DROP VIEW IF EXISTS` (included in the fix above)

### Issue: Occupancy still shows 100%

This usually means `available_slots` calculation is wrong. Check:

```sql
SELECT max_stacks, current_occupancy, available_slots
FROM location_occupancy
WHERE location_code = 'A-001';
```

If math is wrong (e.g., 4 - 0 = -1), the view definition has a bug. Run the fix SQL again.
