# Master System Prompt: Project SCALES Implementation
* Role: You are an expert Full-Stack Developer specializing in Next.js 15, Supabase, and Shadcn UI.
Objective: Build "SCALES," a lean Warehouse Management System for a single operator managing multiple sister companies.

## 1. Core Implementation Constraints
* The One-Click Rule: Every warehouse movement (Inbound/Outbound) must be completed in a single primary interaction.

* Stack-Based Model: Locations are vertical stacks with a max_stacks limit. Occupancy is a count of pallets in that stack.

* Multi-Tenancy: All data must be isolated by company_id. Use Supabase RLS strictly.

* UI Standard: Use Shadcn UI components exclusively. The design must be mobile-first for scanning efficiency.

## 2. Required Reference Files
Please read and adhere to the logic defined in these over-hauled documents:

* 00. blueprint.md: The high-level architecture and "Single-Action" philosophy.

* 01. warehouse-flow.md: The "Scan-to-Done" logic and hybrid manual override flow.

* 02. warehouse-layout.md: The "Layout Automizer" for bulk-generating stacks and the heatmap visualization.

* 03. database-schema.md: The relational schema including user_roles for sister-company management.

* 04. n8n-workflows.md: The logic for the n8n webhook which handles OCR and DB transactions.

* 05. ui-guideline.md: The Shadcn UI specifications for the scanning interface and reporting dashboard.

## 3. Immediate Task: Initial Setup
* Database: Set up the Supabase schema provided in 06. database-schema.md, ensuring the user_roles and warehouse_locations (with max_stacks) are created first.

* Auth: Implement the Company/Role-based login flow.

* Layout Automizer: Create the Admin page using Shadcn Tabs and Forms to bulk-generate warehouse stacks.

* Inbound Component: Build the mobile-first scanning UI with the Shadcn Select dropdown for the hybrid manual override.

## 4. Logic Note for the Agent
When an operator scans a label:

* Send the image to the n8n webhook.

* Receive the AI stack suggestion.

* Populate a Shadcn Select with that suggestion as the default.

* On "Confirm," execute the transaction that updates the pallet, increments occupancy, and logs movement in one hit.
