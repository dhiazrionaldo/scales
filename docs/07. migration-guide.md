# SCALES Database Migration Guide

## Migration: 013_update_locations_grid_model.sql

**Date**: 2026-03-26
**Purpose**: Align `warehouse_locations` table with the 2D Grid Stack Model as defined in the SCALES documentation.

---

## Problem Statement

**Previous Issue**: The `warehouse_locations` table only tracked:
- `max_stacks` was missing (only volume-based capacity)
- No grid positioning for 2D visualization (required for heatmap)
- No company_id for multi-tenant isolation

**Documentation Requirement** (from `rules.md` & `warehouse-layout.md`):
> A "Location" is a vertical stack.
> The warehouse is represented as a 2D Grid of Stacks.

---

## What Changed

### 1. New Columns Added to `warehouse_locations`

| Column | Type | Purpose | Default |
|--------|------|---------|---------|
| `grid_row` | INTEGER | Row position in 2D grid (0-indexed) | NULL |
| `grid_column` | INTEGER | Column position in 2D grid (0-indexed) | NULL |
| `max_stacks` | INTEGER | Vertical capacity (pallets per stack) | 4 |
| `grid_zone` | VARCHAR(10) | Zone prefix (e.g., "A", "B", "C") | NULL |
| `company_id` | UUID | Multi-tenant data isolation | NULL |
| `grid_updated_at` | TIMESTAMP | Audit trail for grid changes | NOW() |

### 2. Unique Constraints Updated

**Old**:
```sql
UNIQUE(warehouse_id, location_code)
```

**New**:
```sql
UNIQUE(company_id, warehouse_id, location_code)
```

This ensures location codes are unique **per company per warehouse** (multi-tenant safety).

### 3. New View: `location_occupancy`

Provides **real-time occupancy tracking** without complex queries:

```sql
SELECT * FROM public.location_occupancy
WHERE warehouse_id = $1 AND occupancy_status = 'PARTIAL';
```

**Columns**:
- `location_id`: The location UUID
- `location_code`: Human-readable code
- `current_occupancy`: Count of pallets with status='STORED'
- `available_slots`: max_stacks - current_occupancy
- `occupancy_status`: 'EMPTY' | 'PARTIAL' | 'FULL'

### 4. New Function: `can_store_pallet(location_id, company_id)`

**Purpose**: Safety check before storing a pallet.

**Usage**:
```sql
-- In a transaction before INSERT into pallets:
SELECT public.can_store_pallet($location_id, $company_id)::boolean;
-- Returns: true if current_occupancy < max_stacks
```

### 5. Indexes Added

For optimal query performance:

```sql
-- Grid-based queries (heatmap generation)
CREATE INDEX idx_warehouse_locations_grid_positioning
ON warehouse_locations(warehouse_id, grid_zone, grid_row, grid_column);

-- Multi-tenant queries
CREATE INDEX idx_warehouse_locations_company ON warehouse_locations(company_id);

-- Status filters
CREATE INDEX idx_warehouse_locations_status
ON warehouse_locations(status) WHERE status != 'DISABLED';

CREATE INDEX idx_warehouse_locations_zone ON warehouse_locations(grid_zone);
```

## Migration Steps (Admin Guide)

### Step 1: Backup Database
```bash
# Using Supabase CLI
supabase db pull  # Creates a local backup of remote schema

# Or via Supabase Dashboard:
# Project Settings → Backups → Create Backup
```

### Step 2: Run Migration
```bash
# Using Supabase CLI
supabase migration up --output 013_update_locations_grid_model.sql

# Or directly in Supabase SQL Editor:
# Copy contents of migrations/013_update_locations_grid_model.sql and execute
```

### Step 3: Data Migration (If Existing Locations)

If you already have locations, populate the new grid fields:

```sql
-- Example: Zone A, 5 rows × 10 columns, max_stacks = 4
UPDATE public.warehouse_locations
SET
  grid_zone = SUBSTRING(zone, 1, 1),
  grid_row = FLOOR((ROW_NUMBER() OVER (PARTITION BY zone ORDER BY location_code) - 1) / 10)::INTEGER,
  grid_column = ((ROW_NUMBER() OVER (PARTITION BY zone ORDER BY location_code) - 1) % 10)::INTEGER,
  max_stacks = 4
WHERE grid_row IS NULL AND grid_column IS NULL;
```

### Step 4: Assign company_id

```sql
-- Link locations to companies via their warehouses
UPDATE public.warehouse_locations wl
SET company_id = w.company_id
FROM public.warehouses w
WHERE wl.warehouse_id = w.id AND wl.company_id IS NULL;

-- Add NOT NULL constraint after data is populated
ALTER TABLE public.warehouse_locations
ALTER COLUMN company_id SET NOT NULL;
```

### Step 5: Verify Migration

```sql
-- Check all locations have grid positioning
SELECT COUNT(*) as total,
       COUNT(CASE WHEN grid_row IS NULL THEN 1 END) as missing_row,
       COUNT(CASE WHEN grid_column IS NULL THEN 1 END) as missing_column
FROM public.warehouse_locations;

-- Should return: total=X, missing_row=0, missing_column=0

-- Check occupancy view works
SELECT * FROM public.location_occupancy LIMIT 5;

-- Check multi-tenant constraint
SELECT COUNT(*) FROM public.warehouse_locations
WHERE company_id IS NULL;
-- Should return: 0
```

## Usage Examples

### 1. Generate a Grid of Stacks (Admin Automizer)

```sql
-- Create a 5×10 grid in Zone A for a warehouse
INSERT INTO warehouse_locations (
  warehouse_id, company_id, location_code,
  grid_zone, grid_row, grid_column, max_stacks, status
)
SELECT
  $warehouse_id,
  $company_id,
  'A-' || LPAD((row_num * 10 + col_num)::text, 3, '0'),
  'A',
  row_num,
  col_num,
  4,
  'AVAILABLE'
FROM (SELECT generate_series(0, 4) AS row_num) r
CROSS JOIN (SELECT generate_series(0, 9) AS col_num) c;

-- Result: 50 locations (A-000 through A-049)
```

### 2. Get Occupancy for Heatmap Visualization

```sql
-- Get all zones with their occupancy status
SELECT
  grid_zone,
  grid_row,
  grid_column,
  location_code,
  occupancy_status,
  current_occupancy,
  max_stacks
FROM public.location_occupancy
WHERE warehouse_id = $warehouse_id
  AND company_id = $company_id
ORDER BY grid_zone, grid_row, grid_column;
```

### 3. Find Empty Stacks for Inbound

```sql
-- Query: Find first available empty stack in Zone A
SELECT id, location_code, grid_row, grid_column
FROM public.location_occupancy
WHERE warehouse_id = $warehouse_id
  AND company_id = $company_id
  AND grid_zone = 'A'
  AND occupancy_status = 'EMPTY'
ORDER BY grid_row, grid_column
LIMIT 1;
```

### 4. Check Before Storing a Pallet

```sql
-- In transaction:
SELECT public.can_store_pallet($location_id, $company_id) AS can_store;

-- If true, proceed with:
INSERT INTO pallets (hu_code, product_name, qty, batch, location_id, company_id, status)
VALUES ($hu_code, $product_name, $qty, $batch, $location_id, $company_id, 'STORED');

-- Insert movement record:
INSERT INTO stock_movements (pallet_id, to_location_id, movement_type, operator_id, company_id)
SELECT id, $location_id, 'INBOUND', $operator_id, $company_id
FROM pallets WHERE hu_code = $hu_code;
```

### 5. Get Current Warehouse Utilization

```sql
-- Dashboard metric: Overall warehouse capacity usage
SELECT
  warehouse_id,
  company_id,
  SUM(max_stacks) AS total_capacity,
  SUM(current_occupancy) AS total_occupied,
  ROUND(100 * SUM(current_occupancy)::NUMERIC / SUM(max_stacks), 2) AS utilization_percent,
  COUNT(CASE WHEN occupancy_status = 'FULL' THEN 1 END) AS full_locations,
  COUNT(CASE WHEN occupancy_status = 'PARTIAL' THEN 1 END) AS partial_locations,
  COUNT(CASE WHEN occupancy_status = 'EMPTY' THEN 1 END) AS empty_locations
FROM public.location_occupancy
WHERE company_id = $company_id
GROUP BY warehouse_id, company_id;
```

## RLS (Row-Level Security) Queries

With the multi-tenant `company_id` field, apply RLS in your app:

```sql
-- Fetch locations only for the authenticated user's company
SELECT * FROM warehouse_locations
WHERE company_id = auth.jwt() ->> 'company_id'
  AND warehouse_id = $warehouse_id;

-- OR if using user_roles table:
SELECT wl.* FROM warehouse_locations wl
JOIN public.user_roles ur ON ur.company_id = wl.company_id
WHERE ur.user_id = auth.uid()
  AND ur.company_id = wl.company_id;
```

```sql
DROP VIEW IF EXISTS public.location_occupancy;

ALTER TABLE public.warehouse_locations
DROP COLUMN IF EXISTS grid_row, grid_column, max_stacks, grid_zone, company_id, grid_updated_at;
```

## Summary

| Aspect | Before | After |
|--------|--------|-------|
| **Grid Support** | No (only volume-based) | Yes (2D row/column) |

| **Multi-Tenancy** | Partial | Full (company_id isolation) |
| **Occupancy Tracking** | Via JOIN to pallets | Real-time VIEW + function |
| **Zone Organization** | In zone column | grid_zone + grid_row/column |
