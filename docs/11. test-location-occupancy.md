# Test: Location Occupancy Display

**Ensure the dropdown correctly shows updated occupancy percentages**

---

## Prerequisites

✅ Database location_occupancy view has been fixed (see `10. fix-location-occupancy-view.md`)
✅ Warehouse locations exist (at least 5 with max_stacks = 4)
✅ Latest code deployed (includes occupancy display improvements)

---

## Test Scenario

### Step 1: Verify Database is Correct

```sql
-- Check database shows correct occupancy
SELECT location_code, current_occupancy, max_stacks, occupancy_status
FROM location_occupancy
WHERE location_code IN ('A-001', 'A-002', 'A-003');
```

**Expected Output** (example):
```
location_code | current_occupancy | max_stacks | occupancy_status
A-001         | 0                 | 4          | EMPTY
A-002         | 0                 | 4          | EMPTY
A-003         | 0                 | 4          | EMPTY
```

### Step 2: Hard Refresh Page

1. Open Receiving page
2. **Hard Refresh**: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
3. Wait for page to fully load

### Step 3: Receive First Pallet

1. Upload/scan a pallet image
2. Wait for OCR to process
3. **Check dropdown capacity display**:
   ```
   ✓ A-001 (100% free)
   ✓ A-002 (100% free)
   ✓ A-003 (100% free)
   ```
4. Select **A-001**
5. Click **Confirm & Store**

### Step 4: Verify Database Updated

```sql
-- Check that pallet was assigned
SELECT location_code, current_occupancy, occupancy_status
FROM location_occupancy
WHERE location_code = 'A-001';
```

**Expected**:
```
location_code | current_occupancy | occupancy_status
A-001         | 1                 | PARTIAL
```

**Or check directly**:
```sql
SELECT COUNT(*) as pallet_count, status
FROM pallets
WHERE location_id = (SELECT id FROM warehouse_locations WHERE location_code = 'A-001')
GROUP BY status;
```

**Expected**:
```
pallet_count | status
1            | PUTAWAY_PENDING
```

### Step 5: Receive Second Pallet (Critical Test)

1. Click **Rescan** button on success screen
2. Upload/scan a DIFFERENT pallet image
3. Wait for OCR
4. **Check dropdown capacity display again**:
   ```
   ▲ A-001 (75% free)    ← Should change from 100% to 75%!
   ✓ A-002 (100% free)
   ✓ A-003 (100% free)
   ```

**If A-001 still shows 100% free**: Cache issue, try hard refresh again

### Step 6: Assign 3 More Pallets to Location

Repeat Steps 3-5 two more times:
- Assign to **A-001** (3rd time) → Should show `(50% free)`
- Assign to **A-001** (4th time) → Should show `(25% free)`
- Assign to **A-001** (5th time) → Should show `(FULL)` or disabled

Final database check:
```sql
SELECT location_code, current_occupancy, max_stacks, occupancy_status
FROM location_occupancy
WHERE location_code = 'A-001';
```

**Expected**:
```
location_code | current_occupancy | max_stacks | occupancy_status
A-001         | 4                 | 4          | FULL
```

## Troubleshooting

### Issue: Dropdown Still Shows 100% Free After Assigning Pallets

**Solution 1: Hard Refresh**
```
Ctrl+Shift+R  (Windows)
Cmd+Shift+R   (Mac)
```

**Solution 2: New Pallet**
- Receive a completely NEW pallet (different HU label)
- Don't click "Rescan" - go through full flow again

**Solution 3: Check Database**
```sql
-- Verify database has correct data
SELECT COUNT(*) FROM location_occupancy
WHERE current_occupancy > 0;

-- Should return > 0 (number of locations with pallets)
```

### Issue: Dropdown Shows Wrong Occupancy Numbers

Check calculation:
```sql
SELECT
  location_code,
  max_stacks,
  current_occupancy,
  ROUND(((max_stacks - current_occupancy)::numeric / max_stacks) * 100, 1) as expected_free_percent
FROM location_occupancy
LIMIT 10;
```

Example:
```
location_code | max_stacks | current_occupancy | expected_free_percent
A-001         | 4          | 1                 | 75.0
A-002         | 4          | 2                 | 50.0
A-003         | 4          | 4                 | 0.0
```

UI should display these percentages.

### Issue: "No locations available" in Dropdown

```sql
-- Check locations exist
SELECT COUNT(*) FROM warehouse_locations
WHERE warehouse_id = 'YOUR_ID' AND company_id = 'YOUR_COMPANY_ID';

-- Should return > 0
```

If 0 locations, create them in Settings → Locations
