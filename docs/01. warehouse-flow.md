# SCALES - High-Velocity Operational Flow
### 1. Overview
This document defines the "Single-Action" logic for SCALES. Every warehouse movement must be completed in one primary interaction to support a single-operator manufacturing environment.

### 2. Inbound Power Action (Scan-to-Store)
* Goal: Take a pallet from production and place it in a stack in one step.

* Trigger: Operator opens the "Inbound" camera interface.

* Action: Operator scans the Handling Unit (HU) label.

* System Logic (Background):

* OCR Extraction: Extract Product, Qty, and Batch.

* Auto-Registration: Create a new record in the pallets table.

* Stack Suggestion: AI finds the best warehouse_location where current_occupancy < max_stacks.

* UI Response: The screen flashes Green and displays: "Store in Stack [A-01]."

* Completion: Operator clicks "Confirm" (or it auto-confirms after 3 seconds).

- Result: Pallet status = STORED. Location occupancy increments by 1.

* Manual Override: If the operator prefers a different stack, they click a Dropdown Menu directly on the suggestion card.

- The dropdown lists available stacks filtered by the same zone or product.

- Selecting a stack from the dropdown immediately updates the target location.

3. Outbound Power Action (Scan-to-Ship)
* Goal: Remove a pallet from a stack and ship it in one step.

* Trigger: Operator scans the Delivery Order (DO) document.

* System Logic (Background): 1.  AI identifies the required Product/Batch. 2.  System highlights the specific Stack on the mini-map.

* Action: Operator goes to the stack and scans the HU label of the pallet.

* Completion: One-click "Ship" button.

* Result: Pallet status = SHIPPED. Location occupancy decrements by 1. Location ID is cleared.

### 4. Warehouse Layout Automizer (Setup Flow)
Goal: Rapidly define the warehouse grid for a new sister company.

Process: 
1.  Admin enters "Zone Name" (e.g., Zone A).
2.  Admin enters "Number of Stacks" (e.g., 50).
3.  Admin enters "Default Max Height" (e.g., 4).

* System Logic: Agent generates 50 records in warehouse_locations automatically.

* Visualization: The layout map instantly renders a grid of 50 squares.

### 5. Reporting & Traceability Flow
* Live Dashboard: Shows "Warehouse Fullness %" (Total Occupancy / Total Max Stacks).

* Traceability: Enter a Batch Number → System lists all Stacks currently holding that batch.

* Audit Trail: Every "Inbound" and "Outbound" action creates a stock_movements entry with a timestamp and user ID.

### 6. Master Configuration Flow (The Setup)
Before any scanning happens, the Admin must complete the Master Setup:

* Select Company: Choose which sister company is being configured.

* Define Layout: Use the "Automizer" to create stacks.

* Input: Zone Name, Number of Stacks, Default Max Height.

* Result: System populates warehouse_locations.
