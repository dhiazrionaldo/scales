# SCALES - Lean Manufacturing WMS
High-Velocity "Single-Action" & "Stack-Based" Architecture
### 1. Executive Summary
SCALES is a lean Warehouse Management System designed for a single operator managing high-volume manufacturing output. It replaces complex logistics workflows with two "Power Actions" and an automated stack-based layout. It is natively multi-tenant for sister-company scaling.

### 2. The "Single-Action" Constraint
* The AI agent must prioritize the One-Click Rule:

* Inbound Power Action: A single scan of an HU label triggers OCR, creates the pallet record, and assigns it to a suggested stack.

* Outbound Power Action: A scan of the Delivery Order (DO) plus a confirmation scan of the HU marks the item as SHIPPED and clears the stack space in one background transaction.

### 3. Warehouse Layout Automizer (The Stack Model)
* Instead of complex 3D mapping, the system uses a Grid of Stacks:

The Model: A "Location" is a vertical stack.

* Master Data: Each stack only needs a LocationCode and MaxStacks (e.g., "A-01" can hold 4 pallets).

* Live Map: A 2D grid visualization (Konva.js) showing real-time occupancy.

- Green: Empty.

- Yellow: Partially filled (e.g., 2/4).

- Red: Full (4/4).

* Auto-Layout: The agent should build a "Bulk Importer" where a user can define 100 stacks in one click.

### 4. Lean Reporting & Visibility
Reporting is focused on Status at a Glance rather than deep analytics:

* Inventory Snapshot: Current stock levels filtered by company_id and warehouse_id.

* Movement Log: A simple audit trail of who moved what and when (Receiving/Shipping events).

* Capacity Alert: A dashboard widget showing total % of warehouse utilization.

* Traceability: One-click search by "Batch Number" to see which stacks contain that specific production run.

### 5. Multi-Tenant Guardrails
All tables (Pallets, Locations, Move Logs) MUST have company_id.

The AI agent MUST apply Row Level Security (RLS) so sister companies only see their own layout and reports.

### 6. Tech Stack for Agents
* UI: Next.js 15 (Mobile-first for the operator).

* Database: Supabase (PostgreSQL).

* Auth: Supabase Auth

* Visuals: Konva.js for the 2D Stack Grid.

* Logic: n8n for "Single-Action" orchestration.

### 7. Master Settings & Multi-Tenancy (NEW)
The AI agent must build a Global Settings Module that controls:

* Company Management: Ability to create and switch between Sister Companies.

* Warehouse Registry: Define multiple warehouses per company.

* System Rules: Global toggle for "Auto-Confirm Inbound" (if on, scanning a label stores it instantly without a second click).

* User Scoping: Ensure users are strictly locked to their assigned company_id.
