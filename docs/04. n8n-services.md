# SCALES - n8n Automation Engine
### 1. Role of n8n
n8n acts as the central orchestrator for the "Single-Action" workflow. The Next.js frontend sends a raw image to an n8n webhook, and n8n handles the OCR, stack logic, and Supabase updates in one pass.

### 2. Inbound "Single-Action" Workflow
* Trigger: POST /webhook/inbound

* Node 1: OCR Service: Calls the self-hosted Python OCR to extract hu_code, product_name, batch, and qty from the image.

* Node 2: Stack Selection Logic: * Query warehouse_locations where current_occupancy < max_stacks.

* Prioritize locations that already contain the same product_name and batch.

* Node 3: Database Transaction:

- Create record in pallets table.

- Increment current_occupancy in warehouse_locations.

- Insert stock_movements log.

* Node 4: Response: Return the location_code to the mobile UI.

# 3. Outbound "Single-Action" Workflow
* Trigger: POST /webhook/outbound

* Node 1: DO Processing: Extract requirements from the Delivery Order image.

* Node 2: Pallet Validation: Compare the scanned HU label against the DO requirements.

* Node 3: Execution:

- Update pallets status to SHIPPED.

- Decrement current_occupancy in warehouse_locations.

- Log the outbound movement in stock_movements.
