# Troubleshooting: Empty Location Dropdown

**Problem**: Storage location dropdown is empty when receiving a pallet

---

## Root Cause Analysis

The dropdown is empty because **no warehouse locations exist in the database**. The system needs locations to suggest before it can populate the dropdown.

### Check Logs First

1. **Open Browser DevTools** (F12)
2. Go to **Console** tab
3. Look for messages like:
   ```
   [RECEIVING] API Response: { suggested_locations_count: 0 }
   [RECEIVING] ⚠️ No locations returned from API. Check if:
   ```

4. **Check Server Logs** (your terminal/deployment logs):
   ```
   [RECEIVING] Found 0 locations, requesting AI suggestions...
   [RECEIVING] No available locations found for warehouse...
   ```

---

## Solution: Create Warehouse Locations

### Option 1: Using the UI (Recommended)

**Step 1: Create Locations via Settings Page**

1. Go to **Settings → Locations**
2. Select a **warehouse** from the dropdown
3. Click **Add Location**
4. Fill in the grid-based form:

```
Location Code: A-001
Zone: A
Grid Row: 0
Grid Column: 0
Max Stack Height: 4 pallets
Status: Available
```

5. Click **Create**
6. Repeat to create 5-10 locations (use zones A, B, C)

**Example Pattern**:
```
Zone A: A-001, A-002, A-003, A-004, A-005 (Row 0, Cols 0-4)
Zone B: B-001, B-002, B-003 (Row 0, Cols 0-2)
```

### Option 2: Bulk Create via API (SQL)

If you have direct database access, run:

```sql
-- Create 15 locations in a 5x3 grid for Zone A
INSERT INTO warehouse_locations (warehouse_id, company_id, location_code, grid_zone, grid_row, grid_column, max_stacks, status)
SELECT
  'YOUR_WAREHOUSE_ID'::uuid,
  'YOUR_COMPANY_ID'::uuid,
  'A-' || LPAD((row_num * 5 + col_num + 1)::text, 3, '0'),
  'A',
  row_num,
  col_num,
  4,
  'AVAILABLE'
FROM (SELECT generate_series(0, 2) AS row_num) r
CROSS JOIN (SELECT generate_series(0, 4) AS col_num) c
ON CONFLICT DO NOTHING;

-- Repeat for Zone B
INSERT INTO warehouse_locations (warehouse_id, company_id, location_code, grid_zone, grid_row, grid_column, max_stacks, status)
SELECT
  'YOUR_WAREHOUSE_ID'::uuid,
  'YOUR_COMPANY_ID'::uuid,
  'B-' || LPAD((row_num * 5 + col_num + 1)::text, 3, '0'),
  'B',
  row_num,
  col_num,
  4,
  'AVAILABLE'
FROM (SELECT generate_series(0, 2) AS row_num) r
CROSS JOIN (SELECT generate_series(0, 4) AS col_num) c
ON CONFLICT DO NOTHING;
```

**Replace**:
- `YOUR_WAREHOUSE_ID` - from warehouses table
- `YOUR_COMPANY_ID` - from companies table

### Option 3: Find IDs via Supabase Dashboard

1. Go to **Supabase Dashboard → SQL Editor**
2. Run these queries to find your IDs:

```sql
-- Find your company
SELECT id, name FROM companies LIMIT 5;

-- Find your warehouse
SELECT id, name FROM warehouses LIMIT 5;
```

3. Copy the IDs and use in the SQL script above

## Common Issues

### Issue 1: "Find Your Warehouse/Company ID"

**Error**: Don't know which warehouse_id or company_id to use

**Solution**:

1. Go to **Settings → Company** to see your company name
2. Go to **Settings → Warehouse** to see your warehouses
3. Use Supabase Dashboard to get the UUIDs (4-step guide above)

### Issue 2: Locations Exist But Still Empty

**Possible cause**: Company ID or Warehouse ID mismatch

**Solution**:

1. Check server logs for:
   ```
   [RECEIVING] No available locations found for warehouse...
   ```

2. Verify in Supabase:
   ```sql
   -- Check if locations exist for YOUR warehouse+company
   SELECT count(*) FROM warehouse_locations
   WHERE warehouse_id = 'YOUR_ID'
   AND company_id = 'YOUR_COMPANY_ID';
   ```

   Should return > 0

### Issue 3: Still No Locations After Creating

1. **Refresh the page** (hard refresh with Ctrl+Shift+R)
2. **Check the database** (Supabase Dashboard):
   - Look at `warehouse_locations` table
   - Verify rows were inserted
   - Check `grid_zone`, `grid_row`, `grid_column` are filled

### Issue 4: API Error When Creating Locations

**Error**: "Failed to create location"

**Solutions**:

- Ensure **Max Stack Height** is a number (e.g., 4, not "4")
- Ensure **Grid Row** and **Grid Column** are numbers
- Ensure **Zone** is a single letter (A, B, C)
- Ensure location code is unique

## Database Schema Check

Run this to verify the schema is correct:

```sql
-- Check warehouse_locations table structure
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'warehouse_locations'
ORDER BY ordinal_position;
```

Should include:
- ✅ `id` (uuid)
- ✅ `warehouse_id` (uuid)
- ✅ `company_id` (uuid)
- ✅ `location_code` (varchar)
- ✅ `grid_zone` (varchar)
- ✅ `grid_row` (integer)
- ✅ `grid_column` (integer)
- ✅ `max_stacks` (integer)
- ✅ `status` (varchar)

If missing `grid_*` or `max_stacks` columns, run the migration:

```bash
# Run pending migrations
supabase migration up
```
