# SCALES - Frontend & Shadcn UI Standards
## 1. Design Philosophy
* The UI must be Mobile-First and Action-Oriented. Because there is only one operator, the interface must prioritize speed and high-contrast feedback.

* The One-Click Rule: No operation should require more than one primary button press after a scan.

* Shadcn UI Standard: Use only Shadcn UI components for consistency and speed of development.

## 2. Core Layout & Components (Shadcn)
The agent must use the following Shadcn components for specific functions:

### A. Global Navigation (Sidebar / Mobile Nav)
- Component: Sheet (for mobile) and NavigationMenu (for desktop).

- Feature: Include a Company Switcher (using Shadcn Select or Popover with a Command list) at the top to toggle between sister companies.

### B. Inbound/Outbound Power Action (Scanning)
- Component: Card to wrap the camera feed.

- Component: Drawer or Dialog to show the AI Suggestion result.

- Feedback: Use Sonner or Toast for instant "Success" or "Error" notifications after a scan.

### C. Layout Automizer (Master Settings)
- Component: Tabs to separate "Zone Creation," "Stack Rules," and "OCR Calibration".

- Component: Form with Input and Button for the Bulk Generator (e.g., "Generate 50 Stacks").

### D. Real-Time Heatmap
- Component: A custom Grid using AspectRatio to keep stacks square.

- Logic: Use Shadcn Tooltip on each stack to show the current batch/product details on hover/tap.

## 3. Screen Templates for the Agent
The "Scan-to-Done" Interface
* Top: Badge showing the active Sister Company and Warehouse.

* Center: Live Camera Feed (OCR Trigger).

* Bottom Overlay: A Shadcn Alert that turns Green when a stack is suggested.

* Text: "Suggested: Stack A-05"

* Action: Button "Confirm & Store" (Auto-focus this button).

* The Reporting Dashboard
Grid: Use Shadcn Card for KPI metrics (Occupancy %, Total Pallets, Shipped Today).

* Table: Use DataTable (TanStack Table) for the Movement Log and Traceability Search.

## 4. UI Logic Guardrails
* Empty State: If no stacks are generated, show a Shadcn EmptyState with a direct link to the Layout Automizer.

* Error State: If a stack is full (current_occupancy >= max_stacks), use the destructive variant of the Badge or Button to prevent storage.

## 5. The "Single-Action" Hybrid Component
To maximize the view and maintain a clean "Manufacturing-Ready" interface, use the following Shadcn UI implementation.

### A. Suggestion & Override Card (Shadcn Card)
* Top Section: Display the scanned HU details (Product, Batch) using Shadcn Badge for high visibility.

* Middle Section (The Selection): * Use a Shadcn Select component (Dropdown).

* Value: Defaults to the AI-suggested location_code.

* Interaction: If the operator wants to override, they tap the SelectTrigger. The SelectContent displays other available stacks with their current capacity (e.g., "A-02 [1/4]").

* Bottom Section: A large Shadcn Button with width: full.

* Label: "Confirm Storage in [Selected Location]".

* Style: Use the default or success variant to make the primary action unmistakable.

### B. Visual Hierarchy Guardrails
* Accessibility: The Select dropdown items must have a large touch target for warehouse operators wearing gloves or moving quickly.

* Real-time Logic: When a user selects a new location from the Shadcn Select dropdown, the "Confirm" button label must update instantly to reflect that choice.

* Capacity Warning: Any stack in the dropdown that is at max_stacks must be disabled (disabled prop in SelectItem) to prevent manual errors.

## 6. Reporting & Dashboard UI
The reporting interface must provide "Instant Visibility" for the Admin and "Movement History" for the Operator.

### A. Key Performance Indicators (Shadcn Card)
* Top Row: Use a 3-column grid of Shadcn Card components.

* Card 1 (Occupancy): A circular progress bar or percentage showing "Warehouse Fullness" (total_occupancy / total_max_stacks).

* Card 2 (Throughput): Total Pallets Inbound vs. Outbound in the last 24 hours.

* Card 3 (Sister Company Status): A dropdown to toggle the dashboard view between different sister companies.

### B. Live Warehouse Heatmap (Visual Report)
* Component: A scrollable Area containing the Grid of Stacks.

* Interaction: Clicking a stack in the heatmap opens a Shadcn Sheet (Side Panel) showing:

* List of all Pallets currently in that stack.

* Product names, Batch numbers, and entry timestamps.

* Remaining vertical capacity (e.g., "Space for 2 more pallets").

### C. Traceability & Audit Logs (Shadcn DataTable)
* Component: Use tanstack/react-table with Shadcn Table styling.

* Filter Bar: A Shadcn Input at the top for "Instant Search" by Batch Number or HU Code.

* Columns: * Timestamp

- Action (Inbound/Outbound)

- Operator Name

- Location Code

- Result: Marks as "Success" with a green Badge.

## 7. Master Settings UI (Management)
To manage the sister companies and layout automation, the agent must use:

### A. Company & User Management (Shadcn Tabs)
* Tab 1: Layout Automizer: Form to bulk-generate stacks (Zone, Count, Max Height).

* Tab 2: Sister Companies: List of companies with an "Edit Settings" button for OCR calibration.

* Tab 3: Roles: A table to assign users as ADMIN or OPERATOR per company.

## 8. UI Logic for Reporting
* Real-time Updates: The dashboard must use Supabase Realtime to update the occupancy cards and heatmap instantly when a "Single-Action" scan occurs.

* Exporting: Provide a Shadcn Button with variant="outline" to download the current table view as a CSV/Excel for manufacturing records.
