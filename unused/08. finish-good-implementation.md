# Finish Good Module - Implementation & Self-Hosted OCR Integration

Complete implementation of the Finish Good Module with integration to the self-hosted Python AI OCR service.

## Overview

The Finish Good Module provides a complete warehouse workflow for:
- **Receiving**: Capture HU labels using OCR
- **Putaway**: Store pallets at optimal locations  
- **Picking**: Fulfill delivery orders from inventory
- **Gate Out**: Release shipments for carrier pickup

## Architecture

```
NextJS Frontend
    ↓ (Image capture)
Finish Good Components (React UI)
    ↓ (POST /api/finish-good/*)
NextJS API Routes
    ↓ (Call OCR service)
Self-Hosted Python OCR Service (VM)
    ↓ (Return extracted data)
API Route Processing
    ↓ (Save to database)
Supabase PostgreSQL
```

## Files Created

### 1. OCR Service Integration (`src/services/ocr.service.ts`)
- **Purpose**: Client-side SDK for calling self-hosted OCR service
- **Features**:
  - `extractHULabelFromFile()` - Upload image file
  - `extractHULabelFromBase64()` - Send base64 encoded image
  - `extractDeliveryOrderFromFile()` - Process DO documents
  - `getImageQuality()` - Validate image before submission
  - Automatic retry logic with exponential backoff
  - Configurable timeout and endpoints

### 2. API Routes (Backend)

#### Receiving Route (`/api/finish-good/receiving`)
```typescript
POST /api/finish-good/receiving
Content-Type: multipart/form-data

Body:
- file: Image of HU label
- warehouse_id: UUID

Response:
{
  "success": true,
  "pallet": {
    "id": "pallet-uuid",
    "hu_label": "HU123456",
    "product_name": "Brake Pad A",
    "qty": 120,
    "batch": "BATCH2025A"
  },
  "suggested_locations": [
    {
      "location_id": "loc-uuid",
      "location_code": "A-03-02-04",
      "capacity_available": "80%",
      "distance_score": 0.9
    }
  ]
}
```

**Process**:
1. Authenticate user via Supabase
2. Call OCR service to extract HU data
3. Validate HU label uniqueness
4. Create `hu_labels` record
5. Create `pallets` record (status: RECEIVED)
6. Create `receiving` audit record
7. Get available warehouse locations
8. Return pallet data with suggestions

#### Putaway Route (`/api/finish-good/putaway`)
```typescript
POST /api/finish-good/putaway
Content-Type: application/json

Body:
{
  "pallet_id": "uuid",
  "location_id": "uuid",
  "warehouse_id": "uuid",
  "confirm_image": "data:image/jpeg;base64,..." // optional
}

Response:
{
  "success": true,
  "pallet": {
    "id": "uuid",
    "status": "STORED"
  },
  "location": {
    "id": "uuid",
    "code": "A-03-02-04"
  }
}
```

**Process**:
1. Verify pallet exists and has RECEIVED status
2. Validate location capacity
3. Create `putaway_tasks` record
4. If confirmation image: Extract HU and verify match
5. Update pallet status to STORED
6. Update warehouse location capacity
7. Create inventory location record

#### Picking Route (`/api/finish-good/picking`)
```typescript
POST /api/finish-good/picking
Content-Type: multipart/form-data

Body:
- delivery_order_image: Image of delivery order
- warehouse_id: UUID

Response:
{
  "success": true,
  "delivery_order": {
    "id": "uuid",
    "document_number": "DO-20260305-001"
  },
  "picking_list": [
    {
      "task_id": "uuid",
      "sequence": 1,
      "location_code": "A-03-02-04",
      "hu_label": "HU123456",
      "product_name": "Brake Pad A",
      "qty_to_pick": 120
    }
  ],
  "total_picks_needed": 1
}
```

**Process**:
1. Extract delivery order information via OCR
2. Create or find `delivery_orders` record
3. Query inventory for required items
4. Find pallet locations containing items
5. Create `picking_tasks` for each pallet
6. Calculate remaining quantity
7. Return optimized picking list

#### Gate Out Route (`/api/finish-good/gate-out`)
```typescript
POST /api/finish-good/gate-out
Content-Type: application/json

Body:
{
  "delivery_order_id": "uuid",
  "warehouse_id": "uuid"
}

Response:
{
  "success": true,
  "delivery_order": {
    "id": "uuid",
    "status": "COMPLETED"
  },
  "shipment": {
    "pallets_shipped": 2,
    "picking_tasks_completed": 2,
    "shipment_date": "2026-03-05T10:30:00Z"
  }
}
```

**Process**:
1. Verify all picking tasks completed
2. Create `outbound` records for each pallet
3. Update pallet status to SHIPPED
4. Update delivery order to COMPLETED
5. Create stock transaction records
6. Update inventory available quantities

## Configuration - Self-Hosted OCR Integration

### Environment Variables

Create `.env.local` in your NextJS project root:

```env
# OCR Service Configuration
NEXT_PUBLIC_OCR_SERVICE_URL=http://your-vm-ip:8000
# or for remote VM: https://ocr.yourcompany.com

# Optional: OCR Service Credentials
OCR_API_KEY=your-api-key-here

# Timeout settings (milliseconds)
OCR_TIMEOUT=30000

# Retry settings
OCR_RETRY_ATTEMPTS=3
```

### Self-Hosted VM Setup

Assuming you have the Python AI OCR service running on a self-hosted VM:

#### 1. **Network Configuration**

**Option A: Local Network (VPN)**
```
VM IP: 192.168.1.100
OCR Service URL: http://192.168.1.100:8000
```

**Option B: Public Domain**
```
Domain: ocr.yourcompany.com
OCR Service URL: https://ocr.yourcompany.com
SSL: Required for production
```

**Option C: SSH Tunnel**
```bash
# Create SSH tunnel
ssh -L 8000:localhost:8000 user@your-vm-ip

# Use local tunnel
OCR Service URL: http://localhost:8000
```

#### 2. **Firewall Rules**

If using public domain, allow only HTTPS:

```bash
# On VM
sudo ufw allow 443/tcp  # HTTPS
sudo ufw allow 8000/tcp # OCR service (internal only)
```

For internal network only:
```bash
# Allow from NextJS server IP only
sudo ufw allow from 192.168.1.50 to any port 8000
```

#### 3. **CORS Configuration**

Update OCR service CORS settings in `python_ai_ocr/main.py`:

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:3000",           # Development
        "https://yourapp.com",              # Production
        "https://app.yourcompany.com",      # Alt domain
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## Database Schema

### Required Tables

```sql
-- HU Labels
CREATE TABLE hu_labels (
  id UUID PRIMARY KEY,
  hu_code VARCHAR(50) NOT NULL,
  company_id UUID NOT NULL,
  warehouse_id UUID NOT NULL,
  product_name VARCHAR(255),
  qty INTEGER,
  net_weight VARCHAR(50),
  batch VARCHAR(50),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Pallets
CREATE TABLE pallets (
  id UUID PRIMARY KEY,
  hu_id UUID NOT NULL REFERENCES hu_labels(id),
  location_id UUID,
  status VARCHAR(50) DEFAULT 'RECEIVED',
  received_at TIMESTAMP,
  stored_at TIMESTAMP
);

-- Receiving Operations
CREATE TABLE receiving (
  id UUID PRIMARY KEY,
  hu_id UUID NOT NULL REFERENCES hu_labels(id),
  operator_id UUID NOT NULL,
  received_time TIMESTAMP,
  status VARCHAR(50)
);

-- Putaway Tasks
CREATE TABLE putaway_tasks (
  id UUID PRIMARY KEY,
  pallet_id UUID NOT NULL REFERENCES pallets(id),
  suggested_location UUID,
  confirmed_location UUID,
  operator_id UUID NOT NULL,
  status VARCHAR(50) DEFAULT 'PENDING',
  completed_at TIMESTAMP
);

-- Warehouse Locations
CREATE TABLE warehouse_locations (
  id UUID PRIMARY KEY,
  warehouse_id UUID NOT NULL,
  location_code VARCHAR(20),
  current_volume DECIMAL(10,2),
  capacity_volume DECIMAL(10,2),
  status VARCHAR(50) DEFAULT 'AVAILABLE'
);

-- Delivery Orders
CREATE TABLE delivery_orders (
  id UUID PRIMARY KEY,
  company_id UUID NOT NULL,
  warehouse_id UUID NOT NULL,
  document_number VARCHAR(100),
  status VARCHAR(50) DEFAULT 'OPEN',
  created_at TIMESTAMP DEFAULT NOW()
);

-- Picking Tasks
CREATE TABLE picking_tasks (
  id UUID PRIMARY KEY,
  delivery_order_id UUID NOT NULL REFERENCES delivery_orders(id),
  pallet_id UUID NOT NULL REFERENCES pallets(id),
  location_id UUID NOT NULL,
  qty INTEGER,
  operator_id UUID NOT NULL,
  status VARCHAR(50) DEFAULT 'PENDING'
);

-- Outbound Records
CREATE TABLE outbound (
  id UUID PRIMARY KEY,
  delivery_order_id UUID NOT NULL REFERENCES delivery_orders(id),
  pallet_id UUID NOT NULL REFERENCES pallets(id),
  shipped_at TIMESTAMP DEFAULT NOW()
);

-- Inventory
CREATE TABLE inventory (
  id UUID PRIMARY KEY,
  company_id UUID NOT NULL,
  warehouse_id UUID NOT NULL,
  product_name VARCHAR(255),
  batch VARCHAR(50),
  total_qty INTEGER,
  available_qty INTEGER,
  reserved_qty INTEGER
);

-- Audit Logs
CREATE TABLE audit_logs (
  id UUID PRIMARY KEY,
  company_id UUID NOT NULL,
  warehouse_id UUID NOT NULL,
  operation_type VARCHAR(100),
  operator_id UUID,
  details JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);
```

## Performance Considerations

### OCR Processing Time
- **Target**: < 300ms per scan
- **Typical**: 150-200ms
- **Includes**: Image preprocessing, OCR, text parsing

### Network Latency
If OCR service is on separate VM:
- Local network (same datacenter): 5-10ms
- Remote VM: 20-50ms
- Internet: 100-500ms

### Recommendations
1. **Keep OCR service and NextJS on same network**
2. **Use HTTP/2 for faster connections**
3. **Implement image compression** before sending
4. **Cache OCR results** for duplicate labels
5. **Use connection pooling** for multiple requests

## Troubleshooting

### OCR Service Unavailable
```typescript
// Check health endpoint
const health = await fetch('${NEXT_PUBLIC_OCR_SERVICE_URL}/health');
if (!health.ok) {
  // Service is down - show error to user
  console.error('OCR service unavailable');
}
```

### Low Confidence Scores
- Check image lighting and focus
- Ensure label text is clearly visible
- Verify label format matches recognized patterns
- Try different angle/distance

### Timeout Errors
- Increase `OCR_TIMEOUT` env variable
- Check network connectivity to VM
- Verify OCR service is responsive
- Check VM CPU/memory usage

### CORS Errors
- Verify `allowed_origins` in OCR service config
- Check browser console for specific error
- Ensure HTTPS certificates are valid (if using SSL)

## Security Best Practices

### 1. Network Security
```
Production Setup:
NextJS App --- VPN/Tunnel --- OCR Service VM
                  (Encrypted)
```

### 2. Authentication
For production, add API key authentication:

```python
# In OCR service
@app.post("/api/v1/ocr/hu-label")
async def ocr_hu_label(request: Request):
    api_key = request.headers.get("X-API-Key")
    if api_key != os.getenv("OCR_API_KEY"):
        raise HTTPException(status_code=401, detail="Unauthorized")
```

```typescript
// In NextJS service
private async post<T>(endpoint: string, ...): Promise<T> {
  fetchOptions.headers = {
    'X-API-Key': process.env.OCR_API_KEY,
    'Content-Type': 'application/json',
  };
}
```

### 3. Rate Limiting
Implement rate limiting in reverse proxy (nginx/CloudFlare):

```nginx
limit_req_zone $binary_remote_addr zone=ocr_limit:10m rate=10r/s;
limit_req zone=ocr_limit burst=20;
```
